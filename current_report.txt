INTRODUCTION
Project Background
Micro-mobility solutions
 such as
 bicycles and electric scooters have emerged as a cornerstone of modern urban transportation and short-distance commuting in Malaysia. While these vehicles offer an eco-friendly transport trend, existing road maintenance often overlooks the specific safety requirements of smaller vehicles. Unlike larger motor vehicles such as cars or motorcycles, micro-mobility 
transportation
 is uniquely vulnerable to road hazards due to their small wheel diameters and frequently rigid suspension systems. For these riders, even a minor pothole can result in an immediate loss of control, mechanical failure, or a "surface-induced accident" that throws the rider off-balance. While standard GPS applications provide effective route guidance, real-time data necessary to warn users about the immediate quality of the road surface. This project proposes an AI-driven solution to bridge this safety gap by providing real-time hazard detection specifically tailored for the micro-mobility community.
Problem Statement
The current landscape of road safety technology is primarily designed for larger motor vehicles, leaving a gap in protection for bicycle and electric scooter riders. The core problems addressed by this project are:
Reactive vs Proactive Tools:
 Existing safety tools, such as standard dashcams, are typically reactive because they only record accidents after they occurred rather than preventing them.
Lack of Real-Time Warning Systems: 
There are currently no proactive, real-time warning system available for micro-mobility users that can identify hazards as they appear.
Speed and Severity Correlation: 
Current road reporting is too slow to protect riders in motion and there is a lack of affordable systems that can calculate the relationship between a vehicle`s speed and the road defect.
Objectives of the Project
This project offers the following objectives:
To develop and train a lightweight Object Detection model (YOLO) capable of identifying potholes in real-time on a mobile device.
To implement a Risk Assessment Algorithm that calculates accident probability based on pothole dimensions and real-time vehicle velocity.
To design a User Interface for a mobile application that delivers immediate, non-intrusive safety warnings via audio and visual cues.
Scope of the Project
The scope of this research is defined by the following boundaries:
Target Users: 
The system is designed to help urban commuters using bicycles and electric scooters.
Platform: 
Development is focused on an Android Mobile application utilizing TensorFlow Lite for on-device Edge Computing to ensure privacy and zero-latency.
Environment:
 The system is intended for use on dedicated bike lanes, urban roads and park pathways during daylight and clear weather conditions.
Technology: 
The project utilizes YOLOv8/v11 Nano architectures, Flutter or Kotlin for app 
development and
 the Google GPS API for real-time speed tracking
Expected Outcomes
At the end of this project, it is expected to deliver:
A trained TFLite model optimized for mobile deployment with high precision and recall for road hazards.
A functional Android application prototype demonstrates real-time pothole detection and dynamic risk-level warnings.
A technical report detailing the mathematical correlation between vehicle speed, pothole size and the safety of the rider.
literature review AND PROJECT METHODOLOGY
Literature Review
Overview of Pothole Detection Techniques
Road hazard detection has traditionally relied on vibration sensors or we call it accelerometers or laser scanning. Research by Kim et al. (2021) explored the use of smartphone accelerometers to detect potholes but this method is reactive. Identifying the hazard only after impact happened. Vision based detection is preferred for micro-mobility because it provides a proactive warning which allows riders to avoid hazards entirely.
Deep Learning for Object Detection (YOLO Series)
The YOLO “You Only Look Once” architecture is a state-of-the-art single shot detector that treats object detection as a regression problem.
YOLOv3 and YOLOv4: 
Earlier versions established the feasibility of road damage detection but often required high=performance GPUs, making them less suitable for mobile deployment such as this project.
YOLOv8 and YOLOv11 Nano:
 
These modern architectures utilize advanced feature extraction and optimized “Nano” models designed to specifically for edge devices. This offers a high balance between mean Average Precision (mAP) and interference speed, which is crucial for real time detection on a moving bicycle or scooter.
O
n-Device Edge Computing and TensorFlow Lite
Achieving a “fully offline” system, on-device edge computing is utilized through TensorFlow Lite or TFLite. This framework allows deep learning models to run directly on a smartphone’s CPU/GPU.
Quantization: 
This technique converts 32-bit floating-point parameters into 8-bit integers which significantly reducing the model size and improve the latency without substantial loss in accuracy.
Privacy and Offline Logic: 
By Processing data locally, user privacy is maintained as no camera feed is uploaded to a cloud server and the system itself remains functional in areas with poor network coverage.
Micro-mobility Safety Constraints
Existing literature focuses heavily on motor vehicles, but micro-mobility such as bicycles and e-scooters have different physical constraints. The small wheel diameters make these vehicles more susceptible to surface induced accidents from hazards that a car might easily travers. A specific risk assessment model requires factors in the relationship between vehicle velocity and pothole severity.

Functional Requirements
Functional Requirements define the core actions the system must perform to meet the project objectives.
Real-Time Detection: 
The system shall capture frames from the camera and detect potholes with at least 15 FPS (Frames Per Second) to ensure safety.
Risk Calculation:
 The application shall retrieve current velocity from the GPS hardware and calculate an accident risk percentage based on the potholes size
 and current vehicle speed.
User Alerting:
 The system shall provide an audible sound such as “beep” and a visual red overlay when the risk index exceeds a predefined safety threshold.
Offline Functionality:
 The entire interface and risk assessment logic must function fully without an active internet connection.
Non-functional Requirements.
Non-functional Requirements specify the performance and quality attributes of the system.
Latency: 
The time between a hazard appearing in the frame and the alert being triggered should not exceed 200ms.
Accuracy:
 The YOLO model should achieve a Mean Average Precision (mAP) of at least 75% for road hazard detection.
Battery Efficiency:
 The application should be optimized to minimize power consumption as GPS and camera usage are battery intensive.
Usability:
 The UI must feature high-contrast elements to remain visible to the rider under direct sunlight.
